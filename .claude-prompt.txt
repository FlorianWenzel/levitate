## Issue

**#9 — API delete log for integration sync**

Float exposes **Delete Log** endpoints so that integrations can detect deletions without full data re-syncs:

- `GET /deleted/tasks` — returns deleted allocation (task) IDs with deletion timestamps  
- `GET /deleted/timeoffs` — returns deleted time-off IDs with deletion timestamps  
- `GET /deleted/logged-time` — returns deleted logged-time IDs with deletion timestamps  

Each endpoint supports **cursor-based pagination** (`cursor` + `limit`, max 500) and returns:

```json
{
  "data": [
    { "id": 987654, "timestamp": "2026-01-09T10:30:00Z" }
  ]
}
```

Response headers include `X-Pagination-Next-Cursor` and `X-Pagination-Has-More`.

Records are **retained for 72 hours** and delivery is at-least-once (the consumer deduplicates by ID).

Sources:
- Float changelog announcement: https://www.float.com/whats-new/changelog-april-24
- Float API reference (Delete Log section): https://developer.float.com/api_reference.html#Delete_Log
- Float Swagger paths (`paths/deleted.yaml`): https://developer.float.com/paths/deleted.yaml

## What's missing in levitate

Levitate has no mechanism for consumers (including the Float import sync) to discover which assignments, time-off entries, or logged-time entries have been deleted. The existing `audit_log` table tracks generic entity changes for human auditing, but it is not designed for integration-driven deletion reconciliation and is not exposed through a public API.

During Float import, levitate currently only creates or updates records; it has no way to remove local assignments or time-off entries that were deleted in Float since the last sync.

## Definition of done

1. **New database table** `deleted_log` (or equivalent) with columns:
   - `id` uuid PRIMARY KEY
   - `entity_type` text NOT NULL (values: `assignment`, `time_off`, `logged_time`)
   - `entity_id` uuid NOT NULL
   - `deleted_at` timestamptz NOT NULL DEFAULT now()
   - 72-hour retention enforced (e.g., via periodic cleanup or Postgres trigger)

2. **Write on delete**: when an `assignment`, `time_off`, or `logged_time` row is deleted, a corresponding row is written to `deleted_log`.

3. **Public API endpoints** (admin scope):
   - `GET /api/deleted/assignments`
   - `GET /api/deleted/time-off`
   - `GET /api/deleted/logged-time`
   - Query params: `cursor` (opaque), `limit` (integer, default 50, max 500)
   - Response shape: `{ "data": [ { "id": "<uuid>", "timestamp": "<iso8601>" } ] }`
   - Pagination headers: `X-Pagination-Next-Cursor`, `X-Pagination-Has-More`

4. **Float import integration**: the Float import handler queries Float's `/deleted/*` endpoints during import and removes matching local records before creating/updating the remaining data.

5. **Database migration** included in `backend/migrations/`.
